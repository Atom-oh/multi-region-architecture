Step 2: us-east-1 배포 (인프라 준비)Deploy (Infra Ready)

us-east-1 리전에 인프라를 배포하지만 아직 트래픽은 전달하지 않습니다. Karpenter NodePool 전략과 EC2NodeClass 구성으로 탄력적인 노드 프로비저닝을 준비합니다.Deploy infrastructure to the us-east-1 region but do not route traffic yet. Prepare elastic node provisioning with Karpenter NodePool strategy and EC2NodeClass configuration.

트래픽 분배Traffic Distribution On-Prem 90% / us-west-2 10% / us-east-1 0% — 인프라만 준비, 트래픽 없음Infrastructure ready, no traffic

6개 NodePool 전략6 NodePool Strategy

Karpenter v1.9 - 워크로드별 NodePool 분리Karpenter v1.9 - Workload-specific NodePool Isolation Infrastructure

Karpenter v1.9는 워크로드 특성에 맞는 인스턴스를 자동으로 선택하고 프로비저닝합니다. 6개의 NodePool로 워크로드를 분리하여 비용 최적화와 성능을 동시에 달성합니다.Karpenter v1.9 automatically selects and provisions instances matching workload characteristics. Workloads are separated into 6 NodePools to achieve both cost optimization and performance.

Higher values are selected first. Example: critical(100) takes priority over batch(30).

General NodePool 정의General NodePool Definition

k8s/infra/karpenter/general-nodepool.yaml

apiVersion: karpenter.sh/v1
kind: NodePool
metadata:
  name: general
spec:
  template:
    metadata:
      labels:
        node-pool: general
    spec:
      nodeClassRef:
        group: karpenter.k8s.aws
        kind: EC2NodeClass
        name: default
      requirements:
        - key: karpenter.sh/capacity-type
          operator: In
          values: [spot, on-demand]
        - key: kubernetes.io/arch
          operator: In
          values: [amd64]
        - key: karpenter.k8s.aws/instance-family
          operator: In
          values: [m6i, m7i, c6i, c7i, r6i, r7i]
        - key: karpenter.k8s.aws/instance-size
          operator: In
          values: [large, xlarge, 2xlarge]
  weight: 70
  limits:
    cpu: "200"
    memory: 400Gi
  disruption:
    consolidationPolicy: WhenEmptyOrUnderutilized
    consolidateAfter: 30syaml
Spot 전략Spot Strategy 현재는 amd64 인스턴스만 사용하지만, arm64(Graviton) 추가 시 비용 최적화가 가능합니다. Spot 인스턴스 중단 시 Karpenter가 자동으로 대체 노드를 프로비저닝하며, consolidateAfter: 30s로 빈 노드를 빠르게 정리합니다.Currently using amd64 instances only, but adding arm64 (Graviton) can further optimize costs. When Spot instances are interrupted, Karpenter automatically provisions replacement nodes, and consolidateAfter: 30s quickly cleans up empty nodes.

EC2NodeClass

EC2NodeClass - AMI, 스토리지, 네트워크 구성EC2NodeClass - AMI, Storage, Network Configuration Infrastructure

EC2NodeClass는 Karpenter가 프로비저닝하는 EC2 인스턴스의 AMI, 스토리지, 네트워크 설정을 정의합니다. 모든 NodePool이 동일한 default EC2NodeClass를 참조합니다.EC2NodeClass defines the AMI, storage, and network settings for EC2 instances provisioned by Karpenter. All NodePools reference the same default EC2NodeClass.

k8s/infra/karpenter/ec2nodeclass.yaml

apiVersion: karpenter.k8s.aws/v1
kind: EC2NodeClass
metadata:
  name: default
spec:
  amiSelectorTerms:
    - alias: al2023@latest      # Amazon Linux 2023 latest
  blockDeviceMappings:
    - deviceName: /dev/xvda
      ebs:
        volumeSize: 100Gi
        volumeType: gp3
        iops: 3000
        throughput: 125
        encrypted: true
        deleteOnTermination: true
  role: multi-region-mall-node-group   # IAM role (global, same for both regions)
  subnetSelectorTerms:
    - tags:
        Tier: private              # Use private subnets only
  securityGroupSelectorTerms:
    - tags:
        karpenter.sh/discovery: multi-region-mall
  tags:
    Environment: production
    ManagedBy: karpenteryaml
