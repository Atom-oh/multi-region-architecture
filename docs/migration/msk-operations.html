MSK 멀티 리전 운영MSK Multi-Region Operations

CCS Connected Car 데이터 파이프라인을 위한 MSK 클러스터 운영 가이드입니다. North America 트래픽 규모에 맞는 사이징, 리전 간 복제, CCDI 데이터 플로우 아키텍처를 다룹니다.MSK cluster operations guide for CCS Connected Car data pipeline. Covers sizing for North America traffic scale, cross-region replication, and CCDI data flow architecture.

MSK 클러스터 사이징MSK Cluster Sizing

MSK 클러스터 사이징 - CCS North America 트래픽MSK Cluster Sizing - CCS North America Traffic Infrastructure

현재 MSK 클러스터는 8개 브로커, 50MB/s 처리량, 6K consumer lag으로 운영 중입니다. Gen2(TCP, 2-3M 차량)와 Gen3(MQTT, 0.6M 차량)에서 4개 CBU (Connected Business Unit)를 통해 데이터가 수집됩니다.Current MSK cluster runs 8 brokers, 50MB/s throughput, with 6K consumer lag. Data is collected via 4 CBUs (Connected Business Units) from Gen2 (TCP, 2-3M vehicles) and Gen3 (MQTT, 0.6M vehicles).

브로커 사이징 공식Broker Sizing Formula

파라미터Parameter값Value설명Description
peak_throughput150 MB/s피크 처리량Peak throughput
replication_factor3복제 계수Replication factor
broker_throughput100 MB/s브로커당 처리량 (kafka.m5.2xlarge)Per-broker throughput (kafka.m5.2xlarge)
safety_margin0.7 (70%)안전 마진Safety margin
계산 공식Calculation Formula
브로커 수 = (피크 처리량 × 복제 계수) / (브로커 처리량 × 안전 마진)Brokers = (peak_throughput × replication_factor) / (broker_throughput × safety_margin)
결과: (150 × 3) / (100 × 0.7) = 450 / 70 ≈ 7 → 8 브로커 (3개 AZ에 분산)Result: (150 × 3) / (100 × 0.7) = 450 / 70 ≈ 7 → 8 brokers (distributed across 3 AZs)

MSK Replicator 구성MSK Replicator Configuration

MSK Replicator - 리전 간 토픽 복제MSK Replicator - Cross-Region Topic Replication Database

MSK Replicator는 us-east-1과 us-west-2 간 토픽 복제를 자동화합니다. Consumer group offset 동기화로 리전 장애 시 다른 리전에서 처리를 이어받을 수 있습니다.MSK Replicator automates topic replication between us-east-1 and us-west-2. Consumer group offset synchronization allows another region to take over processing during regional failures.

Terraform 구성Terraform Configuration

terraform/modules/data/msk/main.tf

resource "aws_msk_replicator" "this" {
  count = var.enable_replicator ? 1 : 0

  replicator_name = "${var.environment}-msk-replicator-${var.region}"

  kafka_cluster {
    amazon_msk_cluster {
      msk_cluster_arn = var.source_cluster_arn
    }
    vpc_config {
      subnet_ids          = var.data_subnet_ids
      security_groups_ids = [var.security_group_id]
    }
  }

  kafka_cluster {
    amazon_msk_cluster {
      msk_cluster_arn = var.target_cluster_arn
    }
    vpc_config {
      subnet_ids          = var.data_subnet_ids
      security_groups_ids = [var.security_group_id]
    }
  }

  replication_info_list {
    source_kafka_cluster_arn = var.source_cluster_arn
    target_kafka_cluster_arn = var.target_cluster_arn
    target_compression_type  = "GZIP"  # optimize cross-region transfer costs

    topic_replication {
      topics_to_replicate = var.replicator_topics

      copy_access_control_lists_for_topics = true
      copy_topic_configurations            = true
      detect_and_copy_new_topics           = true
    }

    consumer_group_replication {
      consumer_groups_to_replicate        = [".*"]
      synchronise_consumer_group_offsets  = true  # key for failover recovery
      detect_and_copy_new_consumer_groups = true
    }
  }

  service_execution_role_arn = aws_iam_role.msk_replicator[0].arn
}hcl

CCDI 데이터 파이프라인CCDI Data Pipeline

CCDI 데이터 파이프라인 아키텍처CCDI Data Pipeline Architecture Architecture

CCDI는 VSS(Vehicle Signal Spec), Location, Driving Information 세 가지 서비스 데이터 타입을 처리합니다. History 데이터는 현재 OFF 상태이며, S3+Athena 기반으로 활성화 예정입니다.CCDI processes three service data types: VSS (Vehicle Signal Spec), Location, and Driving Information. History data is currently OFF and will be enabled using S3+Athena.

Lessons Learned
  • Consumer Group offset 동기화는 cutover 전에 충분히 테스트할 것Thoroughly test consumer group offset sync before cutover
  • Topic configuration (retention, partitions)은 사전에 MSK에 동일하게 생성Pre-create identical topic configurations (retention, partitions) in MSK
  • SASL credential rotation 자동화 필수 (Secrets Manager + Lambda)SASL credential rotation automation required (Secrets Manager + Lambda)
  • MirrorMaker2에서 MSK Replicator 전환 시 일시적 lag 증가 예상Expect temporary lag increase when switching from MirrorMaker2 to MSK Replicator
