Step 2: us-east-1 배포 (인프라 준비)Deploy (Infra Ready)

us-east-1 리전에 인프라를 배포하지만 아직 트래픽은 전달하지 않습니다. Karpenter NodePool 전략과 EC2NodeClass 구성으로 탄력적인 노드 프로비저닝을 준비합니다.

트래픽 분배Traffic Distribution On-Prem 90% / us-west-2 10% / us-east-1 0% — 인프라만 준비, 트래픽 없음

6개 NodePool 전략6 NodePool Strategy

Karpenter v1.9 - 워크로드별 NodePool 분리Karpenter v1.9 - Workload-specific NodePool Isolation Infrastructure

Karpenter v1.9는 워크로드 특성에 맞는 인스턴스를 자동으로 선택하고 프로비저닝합니다. 6개의 NodePool로 워크로드를 분리하여 비용 최적화와 성능을 동시에 달성합니다.

Weight 우선순위 Weight는 Karpenter가 여러 NodePool 중 어디에 Pod를 우선 배치할지 결정하는 우선순위입니다. 값이 높을수록 먼저 선택됩니다. 예: critical(100)이 batch(30)보다 우선합니다.

General NodePool 정의

k8s/infra/karpenter/general-nodepool.yaml

apiVersion: karpenter.sh/v1
kind: NodePool
metadata:
  name: general
spec:
  template:
    metadata:
      labels:
        node-pool: general
    spec:
      nodeClassRef:
        group: karpenter.k8s.aws
        kind: EC2NodeClass
        name: default
      requirements:
        - key: karpenter.sh/capacity-type
          operator: In
          values: [spot, on-demand]
        - key: kubernetes.io/arch
          operator: In
          values: [amd64, arm64]
        - key: karpenter.k8s.aws/instance-family
          operator: In
          values: [m6i, m6g, m7i, m7g, c6i, c6g, c7i, c7g, r6i, r6g, r7i, r7g]
        - key: karpenter.k8s.aws/instance-size
          operator: In
          values: [large, xlarge, 2xlarge]
  weight: 70
  limits:
    cpu: "200"
    memory: 400Gi
  disruption:
    consolidationPolicy: WhenEmptyOrUnderutilized
    consolidateAfter: 30syaml
Spot + Graviton 전략 General NodePool은 amd64/arm64 모두 허용하여 Graviton 인스턴스를 활용합니다. Spot 인스턴스 중단 시 Karpenter가 자동으로 대체 노드를 프로비저닝하며, consolidateAfter: 30s로 빈 노드를 빠르게 정리합니다.

EC2NodeClass

EC2NodeClass - AMI, 스토리지, 네트워크 구성 Infrastructure

EC2NodeClass는 Karpenter가 프로비저닝하는 EC2 인스턴스의 AMI, 스토리지, 네트워크 설정을 정의합니다. 모든 NodePool이 동일한 default EC2NodeClass를 참조합니다.

k8s/infra/karpenter/ec2nodeclass.yaml

apiVersion: karpenter.k8s.aws/v1
kind: EC2NodeClass
metadata:
  name: default
spec:
  amiSelectorTerms:
    - alias: al2023@latest      # Amazon Linux 2023 최신
  blockDeviceMappings:
    - deviceName: /dev/xvda
      ebs:
        volumeSize: 100Gi
        volumeType: gp3
        iops: 3000
        throughput: 125
        encrypted: true
        deleteOnTermination: true
  role: multi-region-mall-node-group   # IAM role (글로벌, 양 리전 동일)
  subnetSelectorTerms:
    - tags:
        Tier: private              # private 서브넷만 사용
  securityGroupSelectorTerms:
    - tags:
        karpenter.sh/discovery: multi-region-mall
  tags:
    Environment: production
    ManagedBy: karpenteryaml
