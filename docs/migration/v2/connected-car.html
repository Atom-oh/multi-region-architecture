Connected Car 서비스와 레이턴시Connected Car Services and Latency

CCS (Connected Car Services)에서 멀티리전은 선택이 아닌 필수입니다. 차량 Telemetry 수집 주기 10초, P99 < 200ms 응답 보장이 핵심입니다.For CCS (Connected Car Services), multi-region is not optional but essential. The key is 10-second vehicle telemetry collection interval and P99 < 200ms response guarantee.

레이턴시 = 안전Latency = Safety 자율주행 보조 기능에서 100ms 지연은 시속 100km 기준 약 2.8m의 차이입니다. 경로 재탐색, 위험 알림, OTA 업데이트 등 모든 커넥티드 기능에서 레이턴시는 안전과 직결됩니다.In autonomous driving assist features, 100ms delay equals about 2.8m difference at 100km/h. Latency is directly linked to safety in all connected features: route recalculation, hazard alerts, OTA updates.

Vehicle Region Routing — CCS North America Network

Route53 latency-based routing으로 차량 위치에 따라 가장 가까운 리전으로 자동 라우팅합니다. 현재 LA Equinix DC가 중앙 허브이지만, AWS 전환 후 2개 리전으로 분산됩니다.Route53 latency-based routing automatically routes to the nearest region based on vehicle location. Currently LA Equinix DC is the central hub, but after AWS migration it will be distributed across 2 regions.

VSS (Vehicle Status Service) 캐싱 패턴

# Critical API - Cache-Aside with Hot VSS Data
async function getVehicleStatus(vehicleId) {
  const cacheKey = `vss:${vehicleId}:status`

  // 1. ElastiCache 조회 (P99 < 1ms)
  let status = await redis.get(cacheKey)
  if (status) return JSON.parse(status)   // Cache HIT

  // 2. Aurora 조회 (P99 ~15ms)
  status = await aurora.query(`SELECT * FROM vehicle_status WHERE ...`)

  // 3. Hot data 캐시 저장 (TTL: 10초 - 텔레메트리 주기와 동일)
  await redis.set(cacheKey, JSON.stringify(status), 'EX', 10)
  return status
}javascript
