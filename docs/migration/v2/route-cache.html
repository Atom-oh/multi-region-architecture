GIS Route Cache 전략GIS Route Cache Strategy

GIS (Global Information Service)는 실시간 내비게이션 서비스입니다. Road Guide + Traffic Info를 조합하여 최적 경로를 제공하며, 리전별 독립 캐시로 비용과 지연을 최적화합니다.GIS (Global Information Service) is a real-time navigation service. It combines Road Guide + Traffic Info to provide optimal routes, optimizing cost and latency with region-independent caching.

GIS Architecture — Road Guide + Traffic Info Architecture

GIS는 실시간 내비게이션 서비스로, 세 가지 핵심 컴포넌트로 구성됩니다. 각 서비스별로 Cache(Redis) + NoSQL(MongoDB) 세트를 구성하는 것이 표준 패턴입니다.GIS is a real-time navigation service composed of three core components. The standard pattern is to configure a Cache(Redis) + NoSQL(MongoDB) set for each service.

Traffic Info Cache 전략Traffic Info Cache Strategy

Traffic Info Cache Strategy Database

Traffic Info와 Road Guide는 특성이 다르므로 TTL을 분리합니다. Geohash 기반 구간 캐싱으로 재사용률을 극대화합니다.Traffic Info and Road Guide have different characteristics, so TTL is separated. Geohash-based segment caching maximizes reuse rate.

Geohash 기반 구간 캐싱Geohash-Based Segment Caching

# Geohash 정밀도 6 = ~1.2km x 0.6km 셀
# 출발지-목적지(OD) 조합 대신 구간별 캐싱 → 재사용률 극대화

Cache Key 구조:
  route:segment:{geohash_from}:{geohash_to}:{traffic_level}

Traffic Level 분리:
  - normal  : 일반 통행
  - heavy   : 혼잡
  - severe  : 심각한 정체

예시:
  route:segment:9q8yyk:9q8yym:normal    # LA 도심 (일반)
  route:segment:9q8yyk:9q8yym:heavy     # LA 도심 (혼잡)
  route:segment:9q8yyk:9q8yym:severe    # LA 도심 (심각)

# 혼잡도 변경 시 해당 traffic level 캐시만 교체
# 전체 무효화 방지 → 캐시 효율성 향상text
Traffic Level 분리의 이점Benefits of Traffic Level Separation 같은 구간이라도 교통 상태(normal/heavy/severe)에 따라 별도 캐시 엔트리를 유지합니다. 혼잡도가 변하면 해당 level의 캐시만 교체하여 불필요한 전체 무효화를 방지합니다.Even for the same segment, separate cache entries are maintained based on traffic state (normal/heavy/severe). When congestion changes, only that level's cache is replaced, preventing unnecessary full invalidation.

리전별 독립 GIS CacheRegion-Independent GIS Cache

Region-Independent GIS Cache Infrastructure

ElastiCache를 리전간 동기화하지 않는 것은 의도적인 설계입니다. GIS 경로 캐시는 지역 특화 데이터이므로 리전별 독립이 더 효율적입니다.Not synchronizing ElastiCache across regions is an intentional design. GIS route cache is region-specific data, so independent per-region operation is more efficient.

# ElastiCache 리전간 동기화 vs 독립 운영

Cross-region 전송 비용:
  동기화: ~$0.02/GB (상당한 비용)
  독립:   $0 (없음)

쓰기 레이턴시:
  동기화: +60ms (cross-region RTT 추가)
  독립:   <1ms (로컬 쓰기)

캐시 효율성:
  동기화: 낮음 (타 리전 데이터 불필요)
  독립:   높음 (지역 특화 데이터만)

장애 격리:
  동기화: 복제 장애 시 양 리전 영향
  독립:   완전 독립, 완전 격리text

   - 인기 경로 기반 우선순위 적용

2. 배포 시 Seed Data:
   - Top-1000 인기 경로 사전 투입
   - 배포 직후 캐시 미스 최소화

3. MSK Event Subscription:
   - traffic.events 토픽 구독
   - road.events 토픽 구독
   - 이벤트 기반 능동적 캐시 갱신text
설계 원칙: 지역 특화Design Principle: Regional Specialization us-west-2는 I-5, I-405, US-101 등 서부 도로에 특화되고, us-east-1은 I-95, I-285, NJ Turnpike 등 동부 도로에 특화됩니다. 교집합(중부 구간)은 차량 이동에 따라 자연스럽게 워밍업됩니다.us-west-2 specializes in western roads like I-5, I-405, US-101, while us-east-1 specializes in eastern roads like I-95, I-285, NJ Turnpike. The intersection (central segments) warms up naturally as vehicles travel.

GIS Cache Invalidation — Event-Driven

GIS Cache Invalidation — Event-Driven Database

GIS Cache는 MSK 이벤트 기반으로 무효화됩니다. 이벤트 유형에 따라 안전 영향도를 분류하고, 그에 맞는 무효화 지연을 적용합니다.GIS Cache is invalidated based on MSK events. Safety impact is classified by event type, and appropriate invalidation delays are applied.

# 이벤트 수신 → 영향 구간 계산 → 캐시 무효화

def invalidate_affected_segments(event):
    # 이벤트 타입별 영향 반경
    radius_map = {
        "ACCIDENT": 5,      # 5km
        "ROAD_CLOSURE": 10, # 10km (우회로 포함)
        "CONSTRUCTION": 2,  # 2km
        "CONGESTION": 0      # 해당 구간만
    }

    radius = radius_map.get(event.type, 1)
    affected = get_segments_in_radius(event.geohash, radius)

    for segment in affected:
        redis.delete(f"route:segment:{segment}:*")python
안전 우선 원칙Safety First Principle 사고, 도로 폐쇄 등 안전에 직접 영향을 주는 이벤트는 5초 이내에 캐시를 무효화합니다. 이는 차량이 위험 구간으로 진입하기 전에 대체 경로를 안내하기 위함입니다.Events that directly affect safety, such as accidents and road closures, invalidate the cache within 5 seconds. This is to guide vehicles to alternative routes before they enter dangerous areas.
