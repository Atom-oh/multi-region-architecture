Step 5: AWS 트래픽 확대Traffic Scale-Up (5:2.5:2.5)

AWS로의 트래픽 비중을 50%로 확대하면서 Observability 스택을 통해 서비스 안정성을 검증합니다. OTel (OpenTelemetry) Collector는 Step 1부터 배포되어 있지만, 트래픽이 50%로 확대되는 이 시점에서 sampling 전략과 모니터링 임계값 튜닝이 중요해지므로 여기서 상세히 다룹니다.

트래픽 분배Traffic Distribution On-Prem 50% / us-west-2 25% / us-east-1 25% — AWS가 절반의 트래픽을 처리

OTel (OpenTelemetry) Collector Pipeline

Tail-based Sampling + 듀얼 Export (Tempo + X-Ray) Observability

OTel Collector는 DaemonSet으로 모든 노드에 배포되며, 수집한 트레이스를 Grafana Tempo와 AWS X-Ray에 동시에 전송합니다. Tail-based sampling으로 중요한 트레이스만 선별하여 비용을 절감합니다.

Sampling 정책

k8s/infra/otel-collector/otel-collector.yaml (발췌)

tail_sampling:
  decision_wait: 10s
  num_traces: 100000
  expected_new_traces_per_sec: 1000
  policies:
    - name: errors-policy
      type: status_code
      status_code:
        status_codes:
          - ERROR              # 에러 트레이스: 100% 수집
    - name: slow-requests-policy
      type: latency
      latency:
        threshold_ms: 500    # 500ms 초과: 100% 수집
    - name: probabilistic-policy
      type: probabilistic
      probabilistic:
        sampling_percentage: 10  # 나머지: 10% 샘플링yaml

Sampling 전략 해설

Policy조건수집률목적
errors-policySTATUS = ERROR100%모든 에러를 놓치지 않음
slow-requestsLatency > 500ms100%성능 병목 분석
probabilistic기본10%비용 최적화
Tail-based vs Head-based Head-based sampling은 트레이스 시작 시점에 결정하므로, 나중에 에러가 발생해도 놓칠 수 있습니다. Tail-based는 전체 트레이스가 완료된 후 판단하므로, 에러/슬로우 트레이스를 100% 캡처합니다. 단, 메모리 사용량이 높아 num_traces: 100000으로 버퍼를 충분히 잡아야 합니다.

Export Pipeline

# OTel Collector 파이프라인 흐름
Pods (OTLP gRPC)
    |
    v
OTel Collector (DaemonSet)
    |--- receivers: otlp (gRPC:4317, HTTP:4318)
    |--- processors: tail_sampling, batch
    |
    |--- exporters:
    |      |--- otlp/tempo   (Grafana Tempo, S3 backend)
    |      |--- awsxray      (AWS X-Ray)
    v
Grafana Dashboard
    |--- Tempo: 분산 트레이싱 상세
    |--- X-Ray: Service Map + 리전간 흐름text

Grafana Tempo + CloudWatch

Observability 스택 구성과 모니터링 대시보드Observability Stack & Monitoring Dashboard Observability

Observability 스택은 3개 계층으로 구성됩니다: Metrics (CloudWatch + Prometheus), Logs (CloudWatch Logs), Traces (Tempo + X-Ray). Grafana에서 3가지 신호를 통합하여 크로스리전 서비스 상태를 모니터링합니다.

Observability 계층

Signal수집저장시각화
MetricsPrometheus / CW AgentCloudWatch MetricsGrafana Dashboard
LogsFluent Bit DaemonSetCloudWatch LogsCloudWatch Logs Insights
TracesOTel CollectorTempo (S3) + X-RayGrafana Tempo / X-Ray Console
