Step 6: On-Prem 종료Shutdown - Full AWS

On-Prem 데이터센터를 완전히 폐쇄하고 모든 트래픽을 AWS로 전환합니다. EC2 Sync Connector와 Direct Connect를 제거하고, latency-based routing으로 양 리전에 50:50 분산합니다.

트래픽 분배Traffic Distribution us-west-2 50% / us-east-1 50% — On-Prem 완전 제거, Full AWS

On-Prem 해제는 데이터 검증 → 트래픽 전환 → 리소스 제거 순서로 진행합니다. 각 단계에서 롤백이 가능한 상태를 유지하며, 최종 확인 후에만 물리적 리소스를 제거합니다.

롤백 안전장치 D+7까지는 On-Prem DC를 대기 상태로 유지합니다. 7일은 주간/주말 트래픽 패턴을 최소 한 사이클 관찰하기 위한 기간입니다. 만약 AWS에서 예상치 못한 문제가 발생하면, CloudFront Origin을 다시 추가하고 EC2 Sync Connector를 재활성화하여 On-Prem으로 롤백할 수 있습니다.
