CCDI 데이터 동기화CCDI Data Synchronization

CCDI(Connected Car Data Integration)의 핵심은 차량 데이터의 허용 지연(SLA)을 정의하는 것입니다. VSS(Vehicle Signal Specification) 데이터 유형별로 다른 복제 전략을 적용합니다.The core of CCDI (Connected Car Data Integration) is defining the allowable lag (SLA) for vehicle data. Different replication strategies are applied based on VSS (Vehicle Signal Specification) data types.

CCDI 복제 지연 SLACCDI Replication Lag SLA

CCDI 데이터 유형별 허용 지연과 모니터링CCDI Per-Data-Type Allowable Lag and Monitoring Database

CCDI는 차량에서 수집되는 다양한 신호(VSS)를 처리합니다. 데이터 유형별 중요도와 허용 지연이 다르며, 이에 맞는 복제 전략을 선택합니다.CCDI processes various signals (VSS) collected from vehicles. Each data type has different importance and allowable lag, and we select appropriate replication strategies accordingly.

CCS 데이터 충돌 해결CCS Data Conflict Resolution

CCDI 충돌 방지 설계와 해결 전략CCDI Conflict Prevention Design and Resolution Strategy Database

CCDI의 차량 텔레메트리 데이터는 본질적으로 append-only입니다. 차량이 데이터를 생성하고 서버가 수신하는 단방향 흐름이므로 충돌이 거의 발생하지 않습니다.CCDI vehicle telemetry data is inherently append-only. Since vehicles generate data and servers receive it in a one-way flow, conflicts rarely occur.

CCDI Append-Only 패턴CCDI Append-Only Pattern

# Vehicle telemetry is write-once
# No UPDATE, only INSERT

INSERT INTO vss_data (
    vin,           # Vehicle Identification Number
    timestamp,     # Immutable event time
    signal_type,
    signal_value
) VALUES (?, ?, ?, ?);

# Primary key: (vin, timestamp, signal_type)
# Same event from same vehicle = same key = idempotenttext

VIN 기반 파티셔닝 (Single-Writer 보장)VIN-based Partitioning (Single-Writer Guarantee)

# VIN determines primary region — Hash(VIN) % 2 → region affinity

VIN: KMHXX00XXXX000001 → hash(0x7A3F...) % 2 = 0 → us-east-1
VIN: KMHXX00XXXX000002 → hash(0x8B2E...) % 2 = 1 → us-west-2text
Benefits
  1. VIN당 Single writer → 충돌 없음Single writer per VIN → no conflicts
  2. 차량 데이터가 한 리전에 집중 → cross-region join 불필요All data for a vehicle in one region → no cross-region joins
  3. 예측 가능한 failover (VIN→리전 매핑이 결정적)Predictable failover (VIN→region mapping is deterministic)
