Step 3: 트래픽 시작Traffic Start (9:1:0)

양 리전 인프라 배포와 동기화가 완료된 상태에서 us-west-2로 10% 트래픽 유입을 시작합니다. EC2 Sync Connector가 On-Prem과 AWS 간 데이터를 실시간으로 동기화합니다.With both regions deployed and sync active, start routing 10% traffic to us-west-2. EC2 Sync Connector synchronizes data between On-Prem and AWS in real-time.

트래픽 분배Traffic Distribution On-Prem 90% / us-west-2 10% / us-east-1 0% — 동기화 확인 후 us-west-2로 소량 트래픽 검증Small traffic validation to us-west-2 after sync verification
쓰기 정책: On-Prem Primary 포워딩Write Policy: On-Prem Primary Forwarding 마이그레이션 기간(Step 3~5) 동안 AWS로 유입되는 트래픽의 모든 쓰기 요청은 On-Prem Primary DB로 포워딩됩니다. AWS 데이터 저장소는 읽기 전용으로 동작하며, On-Prem에서 CDC를 통해 복제된 데이터만 사용합니다. 이렇게 하면 역방향 동기화(AWS→On-Prem) 없이 데이터 일관성이 보장됩니다.During migration (Step 3~5), all write requests from AWS traffic are forwarded to On-Prem Primary DB. AWS data stores operate in read-only mode, using only data replicated from On-Prem via CDC. This ensures data consistency without reverse sync (AWS→On-Prem).

EC2 Sync Connector

On-Prem ↔ AWS CDC 동기화 아키텍처On-Prem ↔ AWS CDC Sync Architecture Database

On-Prem과 AWS 간의 데이터 동기화를 위해 EC2 기반 CDC Connector를 사용합니다. On-Prem DC와 AWS VPC를 Direct Connect로 연결하고, EC2 인스턴스에서 CDC 에이전트를 실행합니다.For data synchronization between On-Prem and AWS, we use an EC2-based CDC Connector. The On-Prem DC and AWS VPC are connected via Direct Connect, and CDC agents run on EC2 instances.

EC2 Sync Connector는 Step 6에서 제거됩니다. On-Prem 트래픽이 0%가 되면 CDC 동기화가 더 이상 필요하지 않으며, 이후에는 AWS 네이티브 리전간 동기화만 활성 상태로 유지됩니다.EC2 Sync Connector is removed in Step 6. Once On-Prem traffic reaches 0%, CDC synchronization is no longer needed, and only AWS native inter-region synchronization remains active thereafter.

Route53 가중치 라우팅Route53 Weighted Routing

Route53 Weighted Routing — 9:1 트래픽 분배Route53 Weighted Routing — 9:1 Traffic Split Network

Route53 weighted routing으로 On-Prem 90%, us-west-2 10% 트래픽 비율을 설정합니다. CloudFront Origin Group은 failover 전용이므로, 가중치 기반 분산은 Route53에서 처리합니다. Health Check로 us-west-2 이상 감지 시 자동으로 On-Prem으로 100% 복귀합니다.Configure Route53 weighted routing with On-Prem 90% and us-west-2 10% traffic ratio. CloudFront Origin Group is for failover only, so weighted distribution is handled by Route53. Health Check automatically routes 100% back to On-Prem if us-west-2 issues are detected.
