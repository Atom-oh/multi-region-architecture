Step 6: On-Prem 종료Shutdown - Full AWS

On-Prem 데이터센터를 완전히 폐쇄하고 모든 트래픽을 AWS로 전환합니다. EC2 Sync Connector와 Direct Connect를 제거하고, latency-based routing으로 양 리전에 50:50 분산합니다.Fully shut down the On-Prem datacenter and shift all traffic to AWS. Remove EC2 Sync Connectors and Direct Connect, distributing 50:50 across both regions via latency-based routing.

트래픽 분배Traffic Distribution us-west-2 50% / us-east-1 50% — On-Prem 완전 제거, Full AWSOn-Prem fully removed, Full AWS
쓰기 정책 전환: AWS 직접 쓰기Write Policy Change: AWS Direct Write On-Prem이 0%가 되면서 CDC 동기화와 EC2 Sync Connector가 제거되고, AWS 데이터 저장소에 직접 쓰기가 시작됩니다. DSQL Active-Active와 Atlas Multi-Region Cluster가 리전간 동기화를 담당합니다.With On-Prem at 0%, CDC sync and EC2 Sync Connector are removed, and direct writes to AWS data stores begin. DSQL Active-Active and Atlas Multi-Region Cluster handle cross-region synchronization.

On-Prem DC 해제 절차와 검증 항목On-Prem DC Decommission Procedure & Verification Operations

On-Prem 해제는 데이터 검증 → 트래픽 전환 → 리소스 제거 순서로 진행합니다. 각 단계에서 롤백이 가능한 상태를 유지하며, 최종 확인 후에만 물리적 리소스를 제거합니다.On-Prem decommission proceeds in order: Data validation → Traffic shift → Resource removal. Maintain rollback capability at each stage, removing physical resources only after final confirmation.

The 7 days allow observation of at least one weekday/weekend traffic cycle. If unexpected issues occur on AWS, add the CloudFront Origin back and reactivate EC2 Sync Connector to rollback to On-Prem.

최종 멀티리전 아키텍처Final Multi-Region Architecture

Full AWS 멀티리전 최종 구성도Full AWS Multi-Region Final Architecture Architecture

On-Prem 해제 후의 최종 아키텍처입니다. 모든 트래픽이 CloudFront → Route53 → NLB → EKS로 흐르며, 양 리전의 데이터스토어가 AWS 네이티브 복제로 동기화됩니다.This is the final architecture after On-Prem decommission. All traffic flows through CloudFront → Route53 → NLB → EKS, with datastores in both regions synchronized via AWS-native replication.
