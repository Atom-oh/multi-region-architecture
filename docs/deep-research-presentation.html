08 / CROSS-CUTTING

Three Systemic Issues

Mock/Stub Architecture Gap

Production-grade infrastructure ($10K+/month) running stub-level services. Event-bus doesn't connect to Kafka. Search doesn't connect to OpenSearch. Authentication is entirely mock.

us-west-2 is Warm Standby, Not Active-Active

DocumentDB writes fail. Valkey writes fail (falls back to in-memory). No DSQL cluster. MSK events are region-local. Reality: warm standby for reads with degraded write capability.

Graceful Degradation = Silent Data Loss

Every service falls back to mock data on failure. Orders "created" with mock inventory. Reviews stored only in memory. No alerts on mock fallback activation.
